# 제품 기획

## 1. 제품 정의

Eazy Chorus는 화음 가이드를 만드는 사람이 MR, 파트별 보컬 음원, 가사 싱크, 파트 범례, 화음 표시를 하나의 프로젝트 파일로 묶어 공유할 수 있게 하는 프론트엔드 전용 웹앱이다.

사용자는 웹사이트에서 `.eazychorus` 파일을 열면 서버 로그인이나 클라우드 저장 없이 동일한 연습 화면을 볼 수 있다.

## 2. 해결하려는 문제

화음 가이드는 보통 다음 자료가 흩어져 전달된다.

- MR 파일
- 메인 보컬 가이드
- 상성부/하성부/코러스 가이드
- 가사 문서
- 파트 설명
- 특정 구간 설명

이 방식은 받는 사람이 어느 시간에 어떤 가사를 부르고, 어떤 파트가 들어오고, 어떤 음원을 들어야 하는지 파악하기 어렵다.

Eazy Chorus는 이 자료를 하나의 프로젝트 파일과 하나의 뷰어로 묶는다.

## 3. v1 제품 원칙

- 가사가 화면의 중심이다.
- 서버에 사용자 데이터를 저장하지 않는다.
- 공유 단위는 링크가 아니라 프로젝트 파일이다.
- 음원은 프로젝트 파일 내부에 포함한다.
- 재생 싱크는 Web Audio API의 공통 clock을 기준으로 맞춘다.
- 편집은 데스크탑에서만 지원한다.
- 보기/연습은 데스크탑과 모바일 모두 지원한다.
- 사용자가 직접 판단하고 지정한 정보만 저장한다. 앱은 화음/파트를 자동 분석하지 않는다.

## 4. 공유 모델

v1 공유 흐름은 다음과 같다.

```txt
작성자
-> 웹앱에서 프로젝트 편집
-> .eazychorus 파일로 내보내기
-> 카카오톡, 디스코드, Google Drive, 메일 등으로 파일 공유
-> 받은 사람은 같은 웹앱에서 파일 열기
-> 동일한 가사/싱크/파트/음원으로 연습
```

서버 저장, 로그인, 실시간 동기화는 v1 범위가 아니다.

## 5. MVP 사용자 흐름

### 5.1 프로젝트 생성

1. 새 프로젝트를 만든다.
2. 곡 제목, 아티스트, 키, BPM, 메모를 입력한다.
3. MR 파일을 추가한다.
4. 파트별 보컬/가이드 음원을 추가한다.
5. 각 파트의 이름, 색상, 설명, 표시 방식을 설정한다.

### 5.2 음원 구성

1. MR은 backing track으로 등록한다.
2. MR을 제외한 음원은 Part에 연결되는 audio variant로 등록한다.
3. 하나의 Part는 여러 audio variant를 가질 수 있다.
4. 예: `메인 보컬 FX`, `메인 보컬 No FX`.

### 5.3 가사 import

1. 사용자가 원본 가사를 붙여넣는다.
2. 앱은 일본어/한글 차음/한국어 해석 형식에서 한글 차음만 추출한다.
3. 영어 단독 줄 예외를 자동 처리한다.
4. 사용자는 좌우 diff 스타일 화면에서 원본과 추출 결과를 비교한다.
5. 사용자가 confirm한 결과만 프로젝트 가사 초안으로 저장한다.

### 5.4 가사 편집과 싱크

1. 확정된 가사를 역할별/파트별 lane으로 나눈다.
2. 재생하면서 Space 또는 마우스 입력으로 다음 가사 시작점을 찍는다.
3. 기본적으로 이전 cue의 끝은 다음 cue의 시작과 같다.
4. 간주나 공백이 필요한 경우 별도 gap 입력으로 현재 cue의 끝을 찍는다.
5. 최종 cue는 `startMs`와 `endMs`를 가진다.

### 5.5 범례와 Part Mark

1. MR 제외 음원들을 기준으로 Part/범례를 구성한다.
2. Part에는 이름, 색상, 설명, audio variant 목록이 있다.
3. 독립 가사 역할인 Main/Sub는 lyric segment의 role로 표현한다.
4. 화음이나 보조 파트 표시는 Part Mark로 표현한다.
5. Part Mark는 가사 일부 또는 전체에 위줄, 밑줄, 형광펜 방식으로 표시된다.

### 5.6 보기/연습

1. 전체 가사를 줄글로 보여준다.
2. 현재 재생 위치에 해당하는 cue를 강조한다.
3. 재생 중 auto-scroll로 현재 cue를 따라간다.
4. 하단 재생바에서 play/pause, seek, mute/solo, 반복재생을 조작한다.
5. 가사를 클릭하면 해당 cue 시작 2초 전부터 재생한다.
6. Part 설명과 믹서는 사이드바 또는 플로팅 패널로 숨긴다.

## 6. v1에서 고정된 결정

- `.eazychorus`는 ZIP 기반 프로젝트 파일이다.
- 내부에는 `project.json`과 `media/*`가 포함된다.
- 음원은 외부 URL이 아니라 프로젝트 파일 내부에 저장한다.
- 음원은 export 단계에서 이미 싱크와 길이를 맞춘 것으로 간주한다.
- 트랙별 offset 보정 UI는 v1에서 제공하지 않는다.
- 다만 future compatibility를 위해 optional `offsetMs` 필드는 허용할 수 있다.
- 편집은 데스크탑 전용이다.
- 모바일은 보기/재생 중심이다.
- Main/Sub lyric role은 글자 색과 스타일로 구분한다.
- Harmony/Part Mark는 위줄/밑줄/형광펜으로 구분한다.
- 가사는 직전/직후만 보여주는 노래방 화면이 아니라 전체 줄글로 보여준다.

## 7. v1에서 하지 않는 것

- 백엔드 API
- 로그인
- 클라우드 저장
- 실시간 협업
- 서버 기반 공유 링크
- Google Drive API 직접 연동
- 오디오 스트리밍 재생
- 자동 화음 분석
- 자동 피치 분석
- 자동 일본어 발음 변환
- DAW 수준의 편집 기능
- 모바일 편집

## 8. 아직 열어둘 항목

- 최종 디자인 톤
- 기본 색상 팔레트
- 모바일 권장 최대 파일 크기
- 지원 음원 포맷의 최종 범위
- 대형 프로젝트 파일의 브라우저별 제한 안내
- OPFS/IndexedDB 기반 임시 저장 사용 여부

## 9. 성공 기준

v1은 다음을 만족해야 한다.

- 사용자는 새 프로젝트를 만들 수 있다.
- MR과 파트 음원을 포함한 `.eazychorus` 파일을 저장할 수 있다.
- `.eazychorus` 파일을 다시 열면 동일한 프로젝트가 복원된다.
- 일본어/차음/해석 형식에서 차음 가사를 추출하고 confirm할 수 있다.
- 파트별 가사를 tap-sync로 맞출 수 있다.
- gap 구간을 지정해 가사가 없는 구간을 만들 수 있다.
- 가사 클릭 시 2초 전부터 재생된다.
- 트랙별 mute/solo/volume 조작이 가능하다.
- Part별 audio variant를 선택할 수 있다.
- Viewer는 전체 줄글 가사를 보여주고 현재 cue를 강조한다.
- Main/Sub lyric role은 글자색/스타일로 구분된다.
- Part Mark는 위줄/밑줄/형광펜으로 표시된다.
